# Sample datasets (reference)

These are **curated reference cards** — small, human-readable examples of every
routable function, meant to show the exact training/eval schema. They are NOT
the training data used to train the shipped models (that is generated by
`scripts/build_cards.py` / `scripts/build_multitool_cards.py` and lives in
`data/raw/`, which is gitignored). Use these to understand the card format,
smoke-test a pipeline change, or quick-train a tiny model.

## Files

- `cards_single_tool.jsonl` — 20 cards, one gold tool per request. The default
  router training shape (`q`, `name`, `args`, `target`). Covers all 7 tools +
  `answer_direct` (no-tool) with varied phrasings.
- `cards_multitool.jsonl` — 8 cards in the **disambiguation** shape: the prompt
  lists the full 8-tool set (`tools` field) but the gold is still a single tool.
  See `wiki/findings/2026-07-14-disambiguation-fixed.md`.

## Schema (one JSON object per line)

```json
{"q": "What is 48 - 5 + 20?", "name": "compute", "args": {"expression": "48 - 5 + 20"}, "target": "TOOL compute {\"expression\": \"48 - 5 + 20\"}\n"}
```

| field   | meaning                                                        |
|---------|----------------------------------------------------------------|
| `q`     | the user request (prompt input)                                |
| `name`  | gold function name (must be in `functions/registry.json`)      |
| `args`  | gold JSON args (or `{}` for no-arg / no-tool calls)            |
| `target`| the training target string (`TOOL <name> <json>\n`)            |
| `tools` | (multi-tool only) the full available-tool list shown to the model |

## Try it

```bash
# quick train a toy model on the single-tool sample (CPU, few epochs)
python scripts/train_router.py --out models/scratch/sample --epochs 20 \
    --data data/samples/cards_single_tool.jsonl --max-len 256

# eval the disambiguation sample
python scripts/eval_multitool.py models/scratch/sample data/samples/cards_multitool.jsonl
```
