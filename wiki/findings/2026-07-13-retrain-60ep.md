## Context
Baseline (30ep, d192/4L/6H, ~2.3M) collapsed into repetition loops; rep_penalty
decode-fix test made it WORSE (route_acc 0.40 → 0.35), proving UNDERTRAINING, not
decode bug. See `2026-07-13-baseline-train.md` + `2026-07-13-rep-penalty-decode-fix.md`.

## Change (single variable: epochs)
- epochs: 30 → **60**. Same arch (d192/4L/6H, ~2.3M), same LR 3e-3, same data (946 cards).
- User explicitly declined: LR bump (instability), bigger model ("feels like cheating"
  / would confound the test). So this is the cleanest capacity-via-time test.

## Prediction / success criterion
- Loss curve was still descending at epoch 30 (0.22). If 60ep lifts route_acc
  materially (say > 0.6), we've found the training floor for this arch.
- If it plateaus near 0.35-0.40, the arch itself is too small to hold 6 tools' grammar
  and we revisit size — but ONLY after this clean epoch test.

## Run
- `python scripts/train_router.py --out models/scratch/baseline-60ep --epochs 60
   --d-model 192 --n-layers 4 --n-heads 6 --data data/raw/cards.jsonl`
- W&B: training run + eval run on same checkpoint (rep_penalty OFF for the eval,
